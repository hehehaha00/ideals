# 当前上下文

当前正在做：深化创意编辑部讨论，让用户选择思考立场和讨论机制，锁定一条观点并围绕它发起有限回应，再把结果送回画布继续发散。

上次停在：工作区返回入口和导图镜头临时保存已修复；本轮为编辑部讨论加入标准/激进/落地/自定义阵容、接力/反驳/站队/交易/极端假设五种机制，并增加“生成对立方向”分支。全量 Vitest 19 文件/370 项、生产构建通过。

关键决定：

- 本轮桌面端优先；手机端不作为验收范围。
- 首页提交后直接进入全屏思维导图，不继续显示输入框、阶段条、高级碰撞和脑洞列表。
- 导图节点可拖动并保存位置；画布支持空白处平移、指针中心缩放、回到中心和适应全部节点。
- 画布增加微缩星图、三档缩放层级、局部星系聚焦、搜索定位、Shift 框选和批量操作。
- 用户可以手动新增节点、折叠分支并撤销重做；历史最多保留 50 步且不跨越 AI 导图边界。
- 重复选择或重复锁定不会占用撤销历史；分支可见性与局部聚焦使用线性索引遍历，减少节点增长后的计算压力。
- 微缩星图按真实视口和全局节点边界求交，低缩放和平移后视口框仍保持准确。
- 导图减少 box，强化锁定状态，并使用分层曲线、分类色和选中路径效果。
- 本地存储升级为版本 2，保存工作区以及孵化脑洞对应的炼化结果。
- AI 响应同时校验请求编号和相关状态修订号，避免旧结果覆盖新操作。
- AI 代理限制允许来源，并增加队列上限和端到端截止时间。
- 工作区存储已升级到 V2；损坏条目会被单独过滤，保存失败会明确提示。
- 请求结果按请求编号和状态修订号双重校验；取消、切换主题和拖动后，旧响应不能覆盖当前成果。
- 本地 AI 代理已加入来源白名单、并发队列上限、429 重试提示和总截止时间。
- 当前本地预览使用 `5193`，`.env.local` 已把该端口加入 `IDEA_APP_ORIGINS`，避免导图请求被来源白名单拒绝。
- 全屏导图支持节点拖动并持久化；锁定节点始终显示锁，未锁节点只在交互时显示控制。
- 导图镜头（缩放、平移）作为工作区临时状态持久化；新导图会重置镜头，旧工作区返回时恢复上次位置。
- 初始节点改为完整环形大屏布局，普通节点默认取消盒子，仅选中和交互节点保留强调。
- 初始导图上限收紧为 28 个分支节点，并使用标签宽度感知的确定性散布布局；同层节点可以自动使用完整安全区。
- 继续发散节点围绕父节点向无限画布外侧散布，并避让已有节点；累计节点不受当前视口边界限制。
- 接入 GSAP 与 `@gsap/react`：初次生成从中心爆发，继续发散从当前节点爆发，重掷收束，碰撞路径汇聚。
- AI 请求和节点爆发收尾期间画布完全锁定；系统减少动态效果可以在运行时即时停止动画。
- 脑洞结果页只展示一篇当前报告，左侧窄栏负责在生成结果间快速切换。
- 结果页“返回导图”固定在视口左上角并使用明确的高对比颜色，滚动报告后仍始终可见。
- 炼化结果使用方向对比表、执行时间线和编辑部批注，不再嵌套卡片。
- 变形方向收进菜单；炼化前主操作为“生成完整报告”，炼化后主操作为“收束推进”。
- 默认路径只展示关键词画布；不要求创建会话、选择方法或完成报告，工作区自动保存。
- 碰撞配方只有在选中至少三个节点并点击碰撞后才出现，六种配方会写入 AI 请求和脑洞来源谱系。
- 灵感谱系自动记录来源、镜头、配方和变形；原图不可用或来源不完整时只展示不可交互摘要。
- 反共识挑战只在单篇脑洞报告中通过“换个立场”打开，五种角色结果以编辑部批注展示。
- “召集讨论”只在单篇脑洞报告中按需出现；每场讨论固定由用户代言人、反常识派、跨界连接者和现实构建者完成三轮结构化讨论，不做无限 AI 群聊。
- 讨论中的灵感火花不会自动污染画布；用户点击“采集到画布”后才生成可继续拖动、锁定和发散的新节点，讨论和采集状态均进入本地持久化。
- 用户每场讨论最多介入三次，每次只触发一轮有限回应；可以追问或不同意某条观点，也可以主动补充，并指定四个固定角色中的一个回应。
- 讨论中始终只保留一条锁定观点；锁定后可从讨论顶部发起针对性回应，切换讨论历史或新开一场时自动解除，避免把回应发到错误场次。
- 编辑部讨论支持标准、激进、落地和自定义阵容；自定义阵容限制为 3-4 个角色，阵容会进入模型提示词并随讨论保存。
- 讨论机制支持接力、反驳、站队、交易和极端假设；机制会明确写入本轮任务，避免角色只是轮流发表观点。
- 讨论结果支持采集为节点、沿方向长成分支和生成对立方向；对立节点在谱系中标记 `opposite`，仍可撤销和继续发散。
- 保守、激进和意外方向只有用户明确选择后才能继续；每次生成 4-6 个带 `discussionOrigin` 的新节点，回到来源导图后可以整体撤销。
- 未炼化报告主操作为“深入验证”，炼化后主操作为“收束推进”；挑战和谱系均为次级入口。

- 前端应用放在 `app/` 目录，避免和现有产品文档、脚本、样张混在一起。
- 第一版只做主题输入、维度词、锁定/重掷、碰撞、变形、收藏和本地持久化。
- 技术栈为 Vite + React + TypeScript + Tailwind CSS + Zustand + Vitest + Playwright。
- AI key 不进入前端打包，通过 `app/server` 本地代理读取环境变量或未提交的 `.env.local`。
- 中转站默认地址为 `https://sub2.congmingai.com`，默认模型为 `gpt-5.5`。
- 当前网络直连中转站会返回 Cloudflare 403；通过 `IDEA_AI_PROXY_URL=http://127.0.0.1:7897` 可以真实请求成功。
- AI 能力统一封装在 `services/ideaApi.ts`；代理、网络或模型失败时不再使用本地 fallback，而是向界面提示 `LLM 有问题：...`。
- 服务端支持 SSE 流式转发、10 分钟内存缓存、上下文压缩、提示词拼接和模型 JSON 结构校验。
- Playwright 使用系统 Chrome 跑桌面和移动端主流程，避免依赖浏览器下载。
- MVP 完成验收路径为：输入主题 -> 生成导图 -> 选节点碰撞 -> 生成脑洞 -> 变形 -> 炼化 -> 放入孵化箱 -> 弹窗多选混合 -> 生成新导图 -> 刷新后仍在。
- 首页不再使用左侧“当前会话/孵化箱”和右侧“变形器”常驻栏；收藏只作为右上角计数和生成后的轻量列表出现。
- 最新视觉方向是单列、少面板、少解释文本，以“大输入框入口”为第一屏主角。
- V2 新增 `BrainstormMap`、`MindNode`、`MindEdge` 数据模型，导图节点分为中心、人群、场景、情绪、物件、结构、限制、远联想。
- 新增 `/api/idea/map` AI 代理路由；AI 失败时显示 LLM 错误，不生成假导图。
- `TopicComposer` 现在触发 `generateMindMap()`；旧的维度词和碰撞台仍保留，但主要由选中导图节点转换而来。
- 导图节点可选中和锁定；“用这些节点碰撞成想法”会把节点转换为六类 `DimensionWord`，继续复用现有脑洞生成能力。
- 导图节点新增 `parentId`，可以追溯从中心主题到当前节点的联想路径。
- `ideaEngine` 新增 `rerollMindMapUnlockedNodes()` 和 `expandMindMapNode()`，用于模拟人类脑暴里的“换刺激”和“继续往外跳”。
- 脑洞卡片新增 `sourcePath` 展示，能看到类似“主题 -> 情绪 -> 远联想 -> 限制”的来源链。
- 新增 `/api/idea/refine` AI 代理路由、`IdeaRefinement` 数据模型和 `IdeaRefinery` 卡片内面板。
- 新增 `/api/idea/mix` AI 代理路由和 `MixedIdeaSeed` 数据模型，用于把孵化箱中 2 到 3 个旧想法混合成新主题。
- `ideaStore` 新增 `refinementsByIdeaId`，每张脑洞卡缓存自己的炼化结果；`loading` 新增 `refine`。
- `ideaStore` 新增 `incubatorOpen`、`incubatorFilter`、`incubatorSelectedIdeaIds`、`lastMixedSeed`，弹窗选择状态不写入 localStorage。
- “放入孵化箱”动作会把当前脑洞加入收藏；孵化箱里的“混合一下”会关闭弹窗、填入混合主题并自动请求新导图。
