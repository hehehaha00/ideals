# 有秩序的灵感宇宙画布设计

## 目标

把思维导图从一次性展示区升级为可以长期承载发散工作的主画布。视觉上保留“灵感宇宙”的沉浸感，操作上补齐定位、整理、批量处理和恢复能力；本轮不加入流程引导和移动端适配。

## 核心体验

- 画布是唯一主舞台，顶部和底部控制层固定，不随世界层移动。
- 远景看整体星系，中景看关键词和关系，近景看完整节点信息。
- 用户可以在空白处平移、滚轮缩放、框选节点、批量操作、搜索定位和折叠分支。
- AI 事件期间画布完全锁定；动画结束后恢复原来的视口和选择状态。
- 新节点保留来源路径和短暂能量尾迹，帮助用户理解“从哪里长出来”。

## 功能范围

### 空间与导航

1. 微缩星图显示全部节点、当前视口矩形、折叠分支数量和当前聚焦节点。
2. 视口状态包含 `panX`、`panY`、`scale`，缩放范围为 0.35 到 2.4。
3. 提供回到中心、适应全部节点和缩放比例提示。
4. 视口只在当前页面会话内保存，不写入 AI 响应，避免旧请求覆盖。

### 节点操作

1. 保留单节点点击选择、锁定和拖动。
2. 空白双击打开轻量新增节点表单；节点必须有非空名称和分类，默认连接当前聚焦节点或中心主题。
3. 空白拖拽框选节点；按住 Shift 可以追加选择，框选结果同步到 Zustand。
4. 多选后提供批量锁定、批量解锁、批量碰撞和清除选择。
5. 节点支持折叠子分支；折叠节点仍保留在数据中，微缩星图显示隐藏数量。
6. 搜索框按标签、分类和来源匹配；命中后镜头移动到节点并显示信标高亮。
7. 所有本地编辑动作进入有限历史栈，支持撤销和重做；AI 生成完成后建立新的历史边界。

### 视觉事件

- 初次生成和继续发散使用现有 GSAP 动效，并把新节点标记为短暂新生状态。
- 局部聚焦只调整视觉强调，不改变节点和边数据。
- 选中节点碰撞时显示路径汇聚和中心撞击；系统锁定期间不允许改变视口或选择。
- 降低动态效果时保留高亮、位置变化和操作反馈，关闭粒子、尾迹和镜头抖动。

## 数据与边界

- `MindNode` 增加 `collapsed?: boolean`，不改变现有持久化版本；读取旧数据默认 `false`。
- 选择、折叠和历史记录属于工作区状态，沿用现有 V2 存储兼容策略。
- 不限制累计节点总量；初次模型响应仍限制 28 个分支节点，单次扩展由模型响应大小和代理队列控制。
- 不引入第三方画布库，沿用 React、SVG、Pointer Events 和 GSAP。

## 验收标准

- 画布在 1280x720 和 1440x900 下无横向溢出，微缩星图和固定控制层不遮挡主节点。
- 可以通过测试验证视口变换、框选、多选批量动作、新增节点、折叠、搜索定位和撤销重做。
- 现有 AI 请求可靠性、节点拖动持久化、碰撞流程和报告页流程全部保持通过。
