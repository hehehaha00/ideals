# 单篇脑洞报告设计

## 目标

把结果页从多张不断展开的脑洞卡片，改成“左侧脑洞导航 + 右侧单篇完整报告”。用户可以快速比较标题，但任何时刻只阅读和操作一个脑洞。

## 视觉论点

像一份正在编辑的创意项目提案：暖黑纸面、橙色批注、清晰的标题层级和大段留白。减少约 70% 的边框和背景块，用分隔线、表格、时间线和引用建立结构。

## 页面结构

- 左侧窄栏固定展示结果数量和脑洞标题；当前项用序号、橙色标记和摘要强调。
- 右侧顶部展示标题、一句话判断、来源路径和收藏状态。
- 正文顺序固定为：为什么值得做、第一版怎么做、目标用户与触发场景、三种落地方向、执行时间线、编辑部批注。
- 未炼化时只保留一个主操作“生成完整报告”。
- 炼化后主操作改为“收束推进”；收藏和继续发散作为次级操作。
- “换个角度”收进一个菜单，不再横排六个同级按钮。

## 组件边界

- `IdeaCardList` 负责结果页两栏布局、导航与当前脑洞选择。
- `IdeaCard` 改为单篇报告页，负责报告头、基础判断、变形菜单和主次操作。
- `IdeaRefinery` 改为无卡片的报告正文：生命力摘要、方向对比表、MVP 时间线和角色批注。
- Zustand 继续管理 `activeIdeaId`、收藏、变形、炼化和下一步动作，不新增页面级状态源。

## 交互

- 点击左侧标题立即切换右侧报告，不滚动或展开旧报告。
- 变形菜单支持键盘聚焦，AI 工作期间禁用。
- 未炼化时“生成完整报告”触发现有 refine 请求。
- 已炼化时“收束推进”写入现有 refinement action；“继续发散”保持次级动作。
- 收藏状态始终在报告头可见。

## 验收

- 页面只渲染一篇完整脑洞报告，不再同时显示多张大卡片。
- 左侧可以切换全部生成结果，当前项具有明确 `aria-current`。
- 炼化前后主操作文字和行为正确。
- 三种方向使用对比表；1 小时、1 天、1 周使用时间线；六个角色反馈使用批注/引用样式。
- 桌面 1280×720 与 1440×900 无横向溢出，主操作保持可见。
