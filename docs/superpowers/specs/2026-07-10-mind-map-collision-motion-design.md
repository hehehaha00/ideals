# 思维碰撞动效与散布布局设计

## 目标

把全屏思维导图从静态节点画布升级为“思维碰撞实验室”：生成过程可感知，节点生成后自然散落，继续发散时从当前节点爆发出新节点，AI 工作期间完全锁定画布交互。

## 视觉论点

暖黑色实验舞台中的能量核与思维轨迹。中心主题像持续工作的反应堆，节点是被碰撞抛出的思维碎片；橙色只用于能量、状态和当前路径。

## 交互与动效

1. 初次生成时显示中心能量核、旋转轨道、漂移粒子和分阶段状态文字。
2. 生成完成后，节点从中心向最终坐标错峰炸开，连线随后显现。
3. 继续发散时，以当前节点为碰撞源显示聚能波和传导路径；新节点从源节点向外爆开。
4. 重掷时，未锁节点先向中心收束并闪烁，再散落到新位置；锁定节点保持稳定。
5. 碰撞生成想法时，选中路径依次点亮并向中心汇聚。
6. `map`、`expand`、`reroll`、`ideas` 期间显示全屏工作遮罩，禁止拖动、选择、锁定、返回首页和打开孵化箱。
7. 支持 `prefers-reduced-motion`：保留状态与遮罩，取消位移、旋转和粒子爆发。

## 布局

- 新增确定性散布算法，以中心为基准，将节点按类别分区、按层级控制距离，并进行多轮矩形碰撞分离。
- 输出坐标必须位于桌面安全区 `x=12..87`、`y=20..80`。
- 相邻节点中心保持最小距离，长标签获得更大的水平间隔。
- 展开节点围绕父节点扇形散布，并再次执行局部碰撞分离。
- 同一输入产生稳定布局，避免刷新后位置跳动；用户拖动后的坐标继续持久化。

## 技术边界

- 使用 `gsap` 和 `@gsap/react`，通过 `useGSAP()` 和 `gsap.context()` 自动清理 React 18 严格模式下的动画。
- GSAP 只操作 `transform`、`opacity` 和 SVG 描边，不负责业务状态。
- Zustand 继续作为唯一业务状态源；交互锁定由 `loading !== "idle"` 决定。
- 粒子为少量 DOM/CSS 元素，不引入 Canvas、WebGL 或持续物理引擎。

## 验收

- 28 个节点在 1280×720 和 1440×900 下无明显堆叠，并避开顶部、底部操作区。
- 初次生成、继续发散、重掷和碰撞均有清晰且不同的工作状态。
- AI 工作期间所有画布交互不可用，完成或失败后恢复。
- 新节点以父节点为起点爆发，动画结束后停在稳定坐标。
- 减少动态效果开启时，流程仍完整可用。
