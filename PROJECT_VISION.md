# 脑洞实验室：项目最终愿想文档

## 一句话愿景

做一个模拟人类发散思维的 AI 脑洞实验室。

它不是普通的“给我 10 个项目灵感”的聊天工具，而是一个能陪用户从词语、隐喻、反转、类比和随机碰撞里长出新想法的创意工作台。

## 为什么要做

很多人不是没有执行力，而是在某个阶段突然失去想法：

- 不知道接下来做什么项目。
- 想做点有意思的东西，但一开口就只会想到常见方向。
- 问 AI 要灵感，得到的往往是已有产品换个名字。
- 太早考虑市场、竞品、商业化，反而把脑洞压扁了。

这个产品要解决的不是“帮用户找到确定需求”，而是“帮用户重新进入会想、敢想、愿意继续想的状态”。

## 核心判断

真正的创意不是凭空出现，也不是理性搜索答案。人类发散思维更像一连串心理动作：

1. 从一个词跳到另一个词。
2. 把不相关的东西放在一起。
3. 把一个领域的结构搬到另一个领域。
4. 故意加入限制，让大脑换路。
5. 把常识反过来。
6. 换一个角色重新看问题。
7. 先放一放，过一段时间再重组。

所以产品不应该只输出答案，而应该展示和放大这个过程。

## 产品定位

脑洞实验室是一个“发散优先、收敛在后”的创意工具。

前半段像玩具，允许怪、允许不完整、允许只有一句隐喻。后半段像工具，把某个有生命力的脑洞炼化成可讨论、可收藏、可继续推进的项目原胚。

## 目标用户

第一批用户可以是：

- 独立开发者：想找下一个项目，但不想只做常规 SaaS。
- 产品经理和设计师：需要做概念发散、工作坊、竞品之外的方向探索。
- 内容创作者：需要栏目、选题、设定、脚本、世界观灵感。
- 游戏和互动产品开发者：需要机制、题材、场景和玩法碰撞。
- 使用 AI 编程的人：能快速实现东西，但需要更好的“做什么”。

## 核心体验

用户进入产品时，不是面对一个空聊天框，而是进入一个创意实验室。

他可以输入一句话、一个烦恼、一个行业、一个喜欢的产品、一个奇怪词，甚至只输入“我不知道做什么”。

AI 先不急着给项目列表，而是把输入拆成一批心理维度词：

- 情绪词：烦、急、羞耻、好奇、无聊、想炫耀、怕出错。
- 行为词：记录、比较、伪装、交换、训练、模拟、收藏、审判。
- 场景词：深夜、会议后、考试前、交付前、通勤、摸鱼、崩溃时。
- 物件词：浏览器、截图、表格、日历、聊天记录、文件夹、地图。
- 结构词：博物馆、法庭、赌场、塔防、拍卖、体检、档案馆、黑匣子。

然后用户可以锁定、重掷、拖拽这些词，让 AI 进行碰撞。

## 人类发散思维模型

### 1. 联想扩散

人类想到一个词，会沿着记忆网络往外跳。

例如：

项目 -> 管理 -> 计划 -> 烂尾 -> 债务 -> 遗迹 -> 博物馆。

好创意通常不在第一层近联想里，而在第三层、第四层的远联想里。

产品要让用户看到联想路径，而不是直接吐出结论。

### 2. 类比迁移

把 A 领域的结构搬到 B 领域。

例如：

- 把游戏里的技能树搬到学习计划。
- 把机场塔台搬到项目管理。
- 把法庭审判搬到代码评审。
- 把博物馆搬到个人数字资产整理。

类比的重点不是表面像，而是结构像。

### 3. 概念融合

把两个原本无关的概念融合成新东西。

例如：

- 日记 + 股票 K 线 = 情绪行情图。
- 浏览器历史 + 博物馆 = 注意力博物馆。
- GitHub issues + 许愿池 = 开源愿望池。
- 个人项目 + 考古 = 项目遗迹馆。

产品要提供一个“碰撞台”，让用户把两个或多个概念硬放在一起。

### 4. 约束变形

创意经常来自限制。

例如：

- 如果这个产品每天只能用 1 分钟？
- 如果它不能联网？
- 如果用户是小学生？
- 如果界面必须伪装成计算器？
- 如果它不能给建议，只能提问？

限制会迫使大脑绕开常规路径。

### 5. 反转

把常识倒过来。

例如：

- 不是帮人开始项目，而是帮人结束项目。
- 不是提高效率，而是故意放慢。
- 不是推荐更多内容，而是推荐少看。
- 不是帮用户做决定，而是帮用户保留犹豫。

反转不一定直接可用，但很容易长出独特气质。

### 6. 角色切换

同一个想法，让不同角色来看，会得到完全不同的变形。

例如：

- 懒人怎么看？
- 老板怎么看？
- 小孩怎么看？
- 毒舌用户怎么看？
- 艺术家怎么看？
- 工程师怎么看？
- 商人怎么看？

产品可以把这些角色做成一张“角色圆桌”。

### 7. 孵化和重组

很多灵感不是当场想出来的，而是在隔了一段时间后重新组合出来的。

产品应该支持收藏半成品脑洞，并在第二天、第三天把旧脑洞重新混合，生成新的方向。

## 产品里的几个房间

### 1. 素材池

用户随便丢东西进来：

- 一句话。
- 一个行业。
- 一个烦恼。
- 一个喜欢的游戏。
- 一个工具。
- 一个奇怪词。
- 一段网页内容。
- 一张截图的描述。

AI 把素材拆成多组可碰撞的词。

### 2. 联想扩散室

对每个词生成三圈联想：

- 近联想：最直接、最常见。
- 中联想：开始偏离常规。
- 远联想：更怪、更有想象空间。

产品要展示“从 A 到 B 是怎么跳过去的”。

### 3. 碰撞台

用户可以选择几组词进行组合：

人群 + 场景 + 情绪 + 物件 + 结构 + 限制。

例如：

独立开发者 + 深夜 + 烂尾焦虑 + GitHub 仓库 + 博物馆 + 不能催用户继续做。

可能得到：

“项目遗迹博物馆”：扫描用户废弃的仓库，把它们变成可浏览的作品遗址。每个项目都有墓志铭、残留价值和可复活零件。

这个阶段先不急着判断有没有市场，只判断它有没有继续展开的生命力。

### 4. 变形器

每个脑洞都可以继续变形：

- 更实用一点。
- 更荒诞一点。
- 更游戏化一点。
- 更像 SaaS。
- 更像浏览器插件。
- 更像 Agent skill。
- 面向学生。
- 面向程序员。
- 面向老板。
- 只保留核心隐喻。

第一版想法通常不完美，但它可以被扭成另一个东西。

### 5. 角色圆桌

同一个脑洞，让不同角色点评：

- 懒人：我为什么会用？
- 毒舌用户：这哪里蠢？
- 产品经理：第一版怎么做？
- 艺术家：它的气质是什么？
- 商人：它怎么收费？
- 工程师：最小实现是什么？

角色圆桌不是为了马上否决想法，而是让想法变成立体的东西。

### 6. 孵化箱

用户收藏几个半成品脑洞。

产品隔一段时间重新混合它们，给出新的组合：

“昨天你收藏了项目遗迹博物馆和注意力行情图。我把它们混了一下，得到一个新方向：个人数字行为考古工具。”

孵化箱模拟的是人类灵感的隔夜重组。

## 第一版最小体验

第一版不需要完整商业化，也不需要复杂账号系统。

只要做出四个能力：

1. 输入一个主题，生成多组心理维度词。
2. 用户锁定或重掷词组，组合成脑洞。
3. 每个脑洞展示联想路径和变形按钮。
4. 收藏脑洞，下一轮继续混合孵化。

## 重要设计原则

### 不要太早评分

一开始就问市场多大、竞品是谁、怎么变现，会杀死发散。

产品要把模式分开：

- 发散模式：允许怪、允许不完整、允许只有一句隐喻。
- 炼化模式：把某个脑洞变成项目、MVP、用户、功能。

### 不是替用户想完

产品不应该让用户被动接收答案，而应该让用户参与挑词、锁词、重掷、变形和收藏。

它的目标不是替用户完成思考，而是让用户重新进入思考。

### 过程比结果重要

用户应该看到：

- 这个词是怎么来的。
- 这个脑洞由哪些词碰出来。
- 它可以怎么变形。
- 它为什么有趣。

这会让用户更容易继续参与，而不是只评价“这个点子好不好”。

### 保留一点荒谬感

发散阶段要允许荒谬。很多真正有生命力的想法，一开始都不太像正经产品。

产品不应该过早把所有想法打磨成标准 SaaS 话术。

## 产品气质

这个产品应该像一个创意实验室，而不是咨询报告生成器。

它可以有一点游戏感、有一点随机性、有一点失控感，但不能变成纯随机笑话机。

理想体验是：

用户一开始觉得“我不知道做什么”，十分钟后变成“这个方向好像有点意思，我想再扭一下”。

## 最终目标

脑洞实验室最终要成为一个帮助人类恢复创造力的 AI 工具。

它不保证每次都生成可赚钱项目，也不假装每个想法都有市场。

它真正要交付的是：

- 更多可继续思考的原胚。
- 更远的联想路径。
- 更大胆的组合。
- 更柔和的创作陪伴。
- 从灵感枯竭到重新想玩的状态。

一句话总结：

先陪人发疯，再帮人落地。
