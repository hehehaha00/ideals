# 脑洞实验室 UI 前端风格设计

## 1. 产品上下文

- 产品：脑洞实验室，一个模拟人类发散思维的 AI 创意工作台。
- 用户：独立开发者、产品经理、设计师、内容创作者、游戏和互动产品开发者。
- 阶段：概念到 MVP。
- 核心任务：输入主题、生成导图、选择节点、碰撞脑洞、变形脑洞、炼化成可执行项目、收藏和孵化。
- 非目标：第一阶段不做企业后台感、不做复杂数据仪表盘、不做营销落地页优先。

## 1.1 竞品调研后的 UI 收束方向

调研对象包括 Miro / FigJam 的 AI 白板、Whimsical / Xmind / MindMeister / Taskade 的 AI 脑图、Google Mixboard / Milanote / mymind 的灵感板、Ideanote 的创新管理和 Napkin 的文本转视觉工具。

关键判断：

- 白板类产品擅长“对象上操作”，但容易把用户带进复杂画布管理。
- AI 脑图类产品擅长“一句话生成结构”，但常停在整理提纲，缺少从空白到需求发现的心理过程。
- 灵感板类产品擅长收藏和重组，但如果没有回到主流程的机制，会变成漂亮的收藏夹。
- 创新管理类产品擅长评估推进，但容易过早企业后台化。
- 本产品的差异不是“也能生成脑图”，而是“从我没想法开始，带用户完成发散、碰撞、炼化和孵化”。

因此 V1 UI 必须收束为：

```text
一个输入框 -> 导图主舞台 -> 选中节点碰撞 -> 脑洞卡 -> 炼化 -> 孵化箱抽屉 -> 混合回导图
```

旧的六类维度词和词组碰撞台不再作为主流程并列展示，只作为“高级词组碰撞”折叠工具保留。

## 2. 风格路线

Autonomy assumptions

- primary style：agentic。
- secondary influence：friendly。
- why：产品是 AI 驱动的创意工作台，需要清晰的任务流、明确的 AI 输出和可操作组件；同时“脑洞实验室”需要比常规 SaaS 更柔和、更有可玩感。
- preserved scope：保留愿想文档里的素材池、联想扩散室、碰撞台、变形器、角色圆桌、孵化箱。
- risks：如果后续转向商业化，可能需要降低 playful 元素，提升专业可信感。
- rollback point：可在第一版视觉评审后切换为更极简或更强品牌感方向。

### 视觉 thesis

像一张温暖的认知实验台：白色纸面、墨色思维路径、橙色行动火花、柔和粉绿标本贴片。

### 内容 plan

- 主工作台：首页即产品，不做营销 hero。
- 首屏：品牌、收藏入口、标题和一个大输入框。
- 发散后：输入区收缩，主区域展示发散思维导图。
- 导图下方：只展示当前阶段最相关的动作和结果。
- 旧维度词和旧碰撞台：折叠到“高级词组碰撞”，用于精修，不与导图主流程争夺注意力。
- 脑洞卡片和炼化器：作为导图碰撞后的下一阶段出现。
- 孵化箱：以右侧抽屉式弹窗出现，承载收藏、筛选、详情和混合回主实验台。

### 交互 thesis

- 词组像可移动的思维标本，选中和锁定有明确反馈。
- 脑洞卡片从碰撞台中生长出来，强调“由哪些词碰撞而成”。
- 变形按钮不是普通操作按钮，而像思考方向的拨杆。
- 炼化器像一场脑内小圆桌，先保留脑洞的生命力，再把它推向可执行版本。

## 3. 设计关键词

- 清醒
- 温柔
- 轻实验室感
- 纸面感
- 可拖拽
- 可继续想
- AI 参与但不喧宾夺主

## 6. 布局系统

### 桌面布局

- 应用背景铺满视口。
- 主内容最大宽度：约 960px。
- 页面边距：20px 到 40px。
- 区块间距：32px 或 48px。
- 主工作台采用单列创意流，避免后台感。
- 发散思维导图是生成后的主视觉工作面，桌面端可使用 SVG 曲线和绝对定位节点。

### 页面结构

```text
输入入口 -> 发散思维导图 -> 选中节点碰撞 -> 脑洞卡片 -> 脑洞炼化器 -> 孵化箱抽屉 -> 混合回实验台
```

### 阶段导航

- 生成前：只显示顶部栏、产品说明和大输入框。
- 生成后：显示轻量阶段条，固定 5 个阶段：`输入`、`发散`、`碰撞`、`脑洞`、`孵化`。
- 阶段条只做定位和状态提示，不做复杂路由。
- 当前阶段由已有数据推断：
  - 无导图：输入。
  - 有导图但无脑洞：发散/碰撞。
  - 有脑洞：脑洞。
  - 打开孵化箱：孵化。

### 密度

MVP 用中等密度。词组区域允许较高密度，脑洞卡片保持呼吸感。

## 7. 组件规范

### App Shell

- 顶部栏：产品名和收藏入口。
- 首屏主体：主题输入。
- 生成后主体：阶段条、发散思维导图、脑洞列表和卡片内炼化面板。
- 旧版词组碰撞：折叠在主舞台下方，标题为“高级词组碰撞”，默认收起。
- 收藏区：右上角计数和生成后的轻量列表，第一版不做常驻左栏。

### 发散思维导图

- 中心节点展示原始主题。
- 分支节点包括人群、场景、情绪、物件、结构、限制、远联想。
- 桌面端使用柔和曲线连接节点，选中路径使用橙色高亮。
- 移动端使用分组彩色标签导图，保证节点清晰可读。
- 节点支持选中和锁定，选中节点可以碰撞成脑洞卡片。
- 节点支持重掷未锁节点，模拟人类脑暴里的“换一批刺激”。
- 当前节点支持继续发散，长出 3 个远联想子节点。
- 节点 hover 要轻微浮起，新增节点用短暂生长动效进入。
- 导图是生成后的主视觉，不要被同屏的大面积词组面板稀释。
- 已选节点栏固定在导图底部，作为碰撞前的唯一主操作面。
- 主按钮保持一个：`用这些节点碰撞成想法`。

### 主题输入

- 输入框应像一张大便签，而不是普通搜索框。
- 高度不低于 96px。
- 支持示例提示：“我想做一个有趣的开发者工具”。
- 主按钮文案：`开始发散`。
- 发散强度使用 segmented control：`轻微 / 正常 / 狂野`。

### 维度词 Chip

- 圆角：999px。
- 高度：32px。
- 左侧可显示小圆点或类别缩写。
- 锁定状态：出现锁图标，边框加深。
- 选中状态：外框使用 `spark-500`。
- 重掷时未锁定词轻微淡出再进入。

### 碰撞台

- 默认不在主流程展开，只放入“高级词组碰撞”。
- 使用纸面托盘式布局。
- 每类词占一个槽位。
- 缺失词时显示虚线占位。
- 当前组合句放在托盘下方，例如：
  - 给“独立开发者”，在“深夜”时，带着“烂尾焦虑”，用“GitHub 仓库”和“博物馆”结构，做一个“不能催用户继续做”的产品。

### 脑洞卡片

- 卡片圆角：8px。
- 只在脑洞卡片使用明显卡片感，其他区块尽量保持纸面布局。
- 顶部展示标题。
- 中部展示一句话解释。
- 底部展示来源词和操作：变形、收藏、展开。
- 卡片内必须展示“由哪些词碰撞而来”。
- 卡片内必须展示“来源路径”，例如：`开发者工具 -> 烂尾焦虑 -> 项目遗迹 -> 博物馆 -> 每天只能用 1 分钟`。

### 变形器

- 使用按钮组，不使用下拉菜单隐藏关键方向。
- 第一版固定 6 个方向：
  - 更实用一点。
  - 更荒诞一点。
  - 更游戏化一点。
  - 更像浏览器插件。
  - 更像 Agent skill。
  - 只保留核心隐喻。

### 脑洞炼化器

- 只在当前选中的脑洞卡片内出现，不做常驻侧栏。
- 入口按钮：`炼化这个脑洞`。
- 结果分为五段：
  - 生命力拆解：目标用户、触发场景、核心情绪、已有替代、最小可玩。
  - 人类视角圆桌：懒人用户、毒舌用户、产品经理、工程师、测试、商人各一句反馈。
  - 三种落地方向：玩具版、工具版、产品版。
  - 执行梯度：1 小时 MVP、1 天 MVP、一周版本。
  - 下一步动作：继续发散、收束推进、放入孵化箱。
- 圆桌角色使用不同浅色标签，保证扫描时能分辨视角。
- `放入孵化箱` 会收藏当前脑洞；其他动作先作为用户选择状态保留。

### 孵化箱弹窗

- 入口在右上角：`孵化箱 n`。
- 孵化箱以右侧抽屉式 modal 出现，不做常驻侧栏和独立页面，主工作台背景轻微压暗。
- 抽屉桌面宽度约 720px 到 900px；移动端全屏底部抽屉化。
- 顶部展示标题、说明、关闭和清空筛选，但不做管理后台式工具栏。
- 筛选标签固定为：全部、未炼化、已炼化、可开工、想混合、今天、本周。
- 中间为紧凑想法卡片池，每张卡片展示标题、summary、来源路径、状态标签和选择按钮。
- 详情区在抽屉内部作为“当前想法”面板展示，不再形成大型左右后台布局。
- 选中 2 到 3 个想法后，底部出现混合栏：`已选 n 个想法` 和 `混合一下`。
- `混合一下` 成功后关闭弹窗，把 AI 返回的混合主题填回主输入框，并自动生成新导图。
- 弹窗内卡片可以密一点，但文字必须可扫读，不做表格后台感。
- 孵化箱不是“收藏管理”，而是“灵感抽屉”：重点是挑选、查看来源、混合回主流程。

## 8. 状态设计

### Loading

- 生成维度词：显示“正在扩散联想”。
- 生成脑洞：显示“正在碰撞这些词”。
- 变形脑洞：显示“正在换一种角度”。
- 炼化脑洞：显示“正在炼化”。

### Empty

- 初始状态：不要空白，展示 3 个可点示例主题。
- 无收藏：提示“先收藏一个有生命力的脑洞，它会在孵化箱里继续发酵。”
- 无词组：提示“输入一个主题，系统会先拆出可碰撞的词。”

### Error

- AI 失败：提示 `LLM 有问题：具体错误`。不要生成本地假内容。
- 本地保存失败：提示“收藏没有保存成功，请稍后再试。”
- 孵化箱为空：提示“在脑洞卡片里点放入孵化箱，它们就会留在这里。”
- 孵化箱混合数量不足：提示“请选择 2 到 3 个想法再混合。”

## 9. 动效规则

- 词组生成：从轻微模糊到清晰，时长 180ms。
- Chip 选中：边框和背景在 120ms 内变化。
- 脑洞卡片出现：从碰撞台下方轻微上浮，时长 220ms。
- 变形结果：旧卡片淡出，新卡片从同一位置替换，保留来源词。
- 炼化结果：在卡片内向下展开，角色标签轻微浮现。
- 孵化箱弹窗：从页面中部轻微上浮，背景只做轻压暗，不要强烈舞台感。
- 减少动态模式下关闭位移，只保留透明度变化。

## 10. 图标规则

- 优先使用 Lucide React。
- 常用图标：
  - Sparkles：开始发散。
  - Lock / Unlock：锁定词。
  - Shuffle：重掷。
  - GitMerge：碰撞。
  - WandSparkles：变形。
  - Brain：炼化。
  - Hammer：执行梯度。
  - Archive：孵化箱。
  - Bookmark：收藏。
  - FlaskConical：实验室。
  - Network：联想路径。
  - UsersRound：角色圆桌。

## 11. 内容语气

语气要短、清楚、有一点陪伴感，但不要卖萌。

推荐文案：

- “先给我一个模糊方向。”
- “锁住这个词。”
- “换一批刺激。”
- “把这些词撞一下。”
- “这个脑洞有点生命力。”
- “明天再让它发酵一下。”

避免文案：

- “释放你的无限创造力。”
- “颠覆世界的商业机会。”
- “AI 赋能全链路创新。”
- “一键生成百万级创业点子。”

## 12. 图片生成用视觉约束

生成 UI 概念图时必须遵守：

- 白色和暖纸色为主，不使用深色科技风。
- 主强调色使用橙色 `#FF5701`。
- 使用柔和粉、薄荷绿、浅蓝、浅紫作为词组贴片色。
- 界面必须像真实产品，不要像营销海报。
- 不要使用大量 KPI 卡片。
- 不要使用紫蓝渐变大背景。
- 不要把页面做成传统企业后台。
- 可以有轻微纸张、贴片、路径线、实验台意象。

## 13. 前端实现建议

- 技术栈：Vite + React + TypeScript + Tailwind CSS。
- 图标：Lucide React。
- 状态：Zustand 管理本地会话、词组、收藏。
- AI 请求：统一放到 `services/ideaApi.ts`。
- AI 失败：由 store 统一展示错误，不做 mock/fallback 兜底。
- 本地存储：第一阶段用 localStorage，后续升级为版本化存储。
- 测试：Playwright 覆盖主流程。

## 14. 验收清单

- 用户第一眼能知道这是一个创意工作台，而不是普通聊天机器人。
- 页面有清晰的输入、导图、选节点碰撞、脑洞、炼化、孵化路径。
- 旧维度词和旧碰撞台不会在生成后直接抢占主流程注意力。
- 词组分类一眼可区分。
- 每张脑洞卡片都能看出来源词。
- 主按钮和主要行动使用橙色。
- 视觉有轻实验室感，但不幼稚。
- 移动端不会出现文字重叠。
- 空状态和错误状态都有下一步引导。
